Some Gems for Rails

This guide lists a few valuable gems for your first Rails Project.

After reading this guide you will

  • know how to install and use a gem
  • know some gems
  • know where to find more gems

1 Installing

Gems are ruby packages, they are the prefered unit of code reuse in ruby. You can install them using the command line tool gem:

gem install devise

In a Rails Project you should let bundler do the work: you just add the gem to the Gemfile, and then run

bundle install

Bundler will resolve version conflicts between the gems for you and install and use a set of gems that will work well with each other.

A word of warning: some gems are not written exclusively in ruby, they may contain C code and use C libraries. From the RubyGems Guides:

Many gems use extensions to wrap libraries that are written in C with a ruby wrapper. Examples include nokogiri which wraps libxml2 and libxslt, pg which is an interface to the PostgreSQL database and the mysql and mysql2 gems which provide an interface to the MySQL database.

To install such a gem you first need to research which libraries are used, how they are named on your linux distribution, and then install the libraries. The last step might look like this:

$dev> ssh my.production.machine
$production> apt-get install libmagick++-dev
$production> gem install rmagick
$production> gem install paperclip

Now that you have installed the gem once by hand you can be sure that it can also be installed by bundler.

You might not have permission to install system wide packages and gems on the production machine. If you can't sudo, you need to ask the system administrator to do it for you.

2 Some Gems

This list is based on Dwellables statistics on the Rails Rumble 2013 and coodbeerstartups "Must Have Gems for Development Machine in Ruby on Rails ".

2.1 Mixins for CSS

You don't want to waste your time writing vendor-prefixes for css3 features. Use a mixin library instead:

http://bourbon.io/

2.2 CSS Grid

Bootstrap is used by a lot of projects. See Kehoe(2013) for help with choosing the right bootstrap gem.

But there is a more sophisticated alternative to bootstrap: Neat uses a semantic grid system:

http://neat.bourbon.io/

2.3 File Upload

Uploading Pictures and displaying them is a common task with two good solutions:

  • gem carrierwave or
  • gem paperclip

Both use the library ImageMagick:

  • gem rmagick image processing with image magick

You should also know how to use ImageMagick on the UNIX commandline:

  • command line tool convert to convert to different formats and
  • command line tool mogrify to resize, blur, crop, draw on, flip, join, ... an image and overwrite it.

2.4 Authentication

  • gem devise for login,
  • gem omniauth to integrate authentication methods.
  • gem cancan if you need roles and permissions sceencast.

2.5 Permalinks

You don't want the database keys to be visible in your URLs? Use Friendly IDs instead:

  • gem friendlyid

2.6 Parsing xml and html

  • gem nokogiri

(this is also used by capybara)

2.7 Parsing Markdown

Letting users enter HTML is a dangerous idea - it's really hard to avoid the security problems. An alternative is to let them enter a simpler markup language, like Markdown. You store the markdown in your database, and convert it to HTML when displayed using a gem:

  • gem redcarpet - see Railscast #272

2.8 Activity Stream

If you want to give users an overview of what happened recently you can use an activity stream, similar to what you see in facebook.

  • gem public_activity

2.9 Admin Backend

Create a Backend for manipulating data with a few lines of code:

  • gem activeadmin

2.10 Sending Mail

To send mail from Rails use ActionMailer.

To see the generated mails in your web browser instead of sending them, use the gem letter_opener

2.11 HTTP Requests

This might be handy for downloading from Webpages or APIs

  • gem curb - the http library curl for ruby

2.12 Using APIs

  • gem octokit - for github
  • gem twitter
  • gem gravatar
  • gem koala - for facebook
  • gem barometer - A multi API consuming weather forecasting superstar
  • gem gmaps4rails

2.13 Testing

  • gem factory_girl for creating test data.
  • gem capybara as the "browser" for acceptance tests, with
  • gem poltergeist for testing client side javascript.

2.14 Understanding your Code better

  • gem better_errors

  • gem meta_request - supporting gem for Rails Panel, a Google Chrome extension for Rails development

  • gem annotate - inserts the db schema as comments in your model.rb file

  • gem bullet - helps you improve your usage of activerecord queries

  • gem flay - finds structural similarities in your code, so you can refactor

  • gem rails_best_practices
