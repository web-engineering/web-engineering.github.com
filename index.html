About this Book

This is a textbook for the second year of a computer science or "media informatics" degree program. It builds on Web Development (which is only available in german).

This book is not a self-contained work: There are great resources for learning about the tools and technologies of web engineering - this book tries to use these resources, and bind them together into a coherent study plan.

It was written and published by Brigitte Jellinek in the hope that it will be useful to lecturers and students in educational institutions as well as to self-directed learners on the internet.

Why Web Engineering

Is there really a need for "Web Engineering" as a separate subject? Doesn't "Software Engineering" cover Web Development also?

I would argue that web development has reached a level of complexity where it makes sense to study it in depth, in it's own right. Web projects have some features in common, that separate them from other software projects:

  • different people: in web projects there are many non-programmers involved: designers, usability-experts, community managers, ... more so than in other software development projects.
  • different languages: for web development dynamic languages are used. this implies different design patterns than strongly typed languages
  • no software in boxes: web apps are not delivered once a year in a box. they have to be maintained and updated all the time, continous delivery is needed!

This textbook will try to teach you about general good software engineering practices while always sticking to examples that are actually useful in web development today.
